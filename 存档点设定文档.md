# 存档点设定.md

本文档说明如何在当前 `O---O` 项目中加入一种新的道具：

```text
存档点 Checkpoint
```

存档点不新建独立实体类，而是复用现有的：

```python
Item
```

通过：

```python
prefab = "checkpoint"
effect = "Checkpoint"
trigger_condition = "OnTouch" / "OnAnchor"
```

来表达。

---

# 1. 功能目标

## 1.1 存档点是什么？

存档点是地图中的一种特殊道具。

当玩家触碰或抓住存档点时，它会记录当前关卡状态。

记录内容包括：

```text
1. 棍子的状态
2. 当前地图状态
3. 脆弱墙壁是否已经碎裂
4. 移动墙壁当前位置和移动进度
5. 不稳定墙壁当前倾斜状态
6. 实线/虚线墙状态
7. 岩浆当前高度
8. 已经被消耗的道具状态
9. 已触发的钥匙机关状态
10. 相机位置等可选状态
```

之后如果玩家：

```text
死亡
```

或者按下：

```text
R
```

则不是从关卡开头重新开始，而是从最近激活的存档点恢复。

如果玩家按下：

```text
ESC
```

则从关卡最开始重新开始，忽略存档点。

---

# 2. 设计原则

## 2.1 不新建 Checkpoint 类

存档点应该复用现有的：

```python
Item
```

不要新建：

```python
class Checkpoint:
    pass
```

原因：

```text
1. 存档点本质上是地图中的可触发对象。
2. 当前 Item 已经支持位置、尺寸、prefab、effect、trigger_condition。
3. 存档点可以作为一种特殊 effect。
4. 这样 Tiled 中也可以继续通过 Items 图层拖拽配置。
```

---

## 2.2 只允许一个激活存档点

玩家同时只能有一个有效存档点。

规则：

```text
1. 玩家第一次触发 checkpoint_A：
   checkpoint_A 成为当前激活存档点。

2. 玩家之后触发 checkpoint_B：
   checkpoint_B 成为新的激活存档点。
   checkpoint_A 的存档记录失效。

3. 死亡或按 R：
   从当前激活存档点恢复。

4. 如果没有激活过任何存档点：
   死亡或按 R 时从关卡开头重新开始。
```

---

## 2.3 ESC 始终从头开始

按下：

```text
ESC
```

表示放弃存档点，从关卡初始状态重新开始。

即：

```text
ESC 不读取 checkpoint snapshot
ESC 重新加载原始 level.json / 初始关卡状态
```

---

# 3. 操作规则

## 3.1 死亡时

玩家死亡原因可能包括：

```text
1. 被岩浆追上
2. 掉出地图
3. 其他死亡判定
```

死亡后恢复逻辑：

```text
如果存在激活存档点：
    从存档点 snapshot 恢复
否则：
    从关卡开头重新开始
```

---

## 3.2 按 R 时

按下：

```text
R
```

表示：

```text
从当前存档点重新开始
```

逻辑：

```text
如果存在激活存档点：
    恢复存档点 snapshot
否则：
    从关卡开头重新开始
```

---

## 3.3 按 ESC 时

按下：

```text
ESC
```

表示：

```text
从关卡开头重新开始
```

逻辑：

```text
清空当前 checkpoint snapshot
重新加载关卡初始状态
```

---

## 3.4 触发存档点

存档点的触发方式可以有两种。

### 方式一：触碰触发

```text
trigger_condition = OnTouch
```

表示：

```text
棍子任意部分碰到存档点就触发
```

适合做较大的存档区域。

---

### 方式二：锚定触发

```text
trigger_condition = OnAnchor
```

表示：

```text
棍子端点抓住存档点才触发
```

适合做“必须用一端抓住存档点”的精确玩法。

---

# 4. 推荐最终规则

对于本项目，推荐存档点采用：

```text
trigger_condition = OnTouch
```

原因：

```text
1. 存档点是安全系统，不应该过度苛刻。
2. 玩家只要经过即可存档，体验更顺滑。
3. 可以避免玩家明明经过了存档点却因为没有锚定而没记录。
```

但如果你希望存档点更像“钉住存档”，也可以使用：

```text
trigger_condition = OnAnchor
```

# 5. 存档点在 Item 中的配置

`Item` 已经有这些属性：

```python
self.id
self.x
self.y
self.width
self.height
self.effect
self.value
self.trigger_condition
self.active
self.appearance
self.prefab
```

建议新增或使用以下配置：

```python
prefab = "checkpoint"
effect = "Checkpoint"
trigger_condition = "OnTouch"
consume_on_trigger = False
checkpoint_id = "checkpoint_001"
```

---

## 5.1 item.py 建议新增属性

在 `Item.__init__()` 中加入：

```python
# 存档点 ID。
# 当 prefab="checkpoint" 或 effect="Checkpoint" 时使用。
self.checkpoint_id = config.get("checkpoint_id", self.id)

# 存档点触发后不应该消失。
# checkpoint 默认 consume_on_trigger=False。
if self.effect == "Checkpoint" or self.prefab == "checkpoint":
    self.consume_on_trigger = config.get("consume_on_trigger", False)
else:
    self.consume_on_trigger = config.get("consume_on_trigger", True)
```

注意：

你现在 `Item` 已经有：

```python
self.consume_on_trigger = config.get("consume_on_trigger", True)
```

建议改成上面的逻辑，使 checkpoint 默认不消失。

# 7. 存档数据 snapshot

存档点触发时，不是只记录玩家位置，而是要记录：

```text
整个关卡当前运行状态
```

称为：

```python
checkpoint_snapshot
```

推荐结构：

```python
checkpoint_snapshot = {
    "checkpoint_id": "checkpoint_001",

    "stick": {...},

    "walls": {...},

    "items": {...},

    "hazards": {...},

    "lava": {...},

    "camera": {...},

    "level_runtime": {...},
}
```

# 9. 棍子状态记录

必须记录棍子完整状态。

例如：

```python
"stick": {
    "length": stick.length,
    "angular_speed": stick.angular_speed,
    "rotation_direction": stick.rotation_direction,

    "angle": stick.angle,
    "center_x": stick.center_x,
    "center_y": stick.center_y,
    "velocity_x": stick.velocity_x,
    "velocity_y": stick.velocity_y,

    "state": stick.state,
    "anchor_side": stick.anchor_side,
    "anchor_x": stick.anchor_x,
    "anchor_y": stick.anchor_y,

    "attached_wall_id": stick.attached_wall.id if stick.attached_wall else None,
    "anchor_local_pos": [
        stick.anchor_local_pos.x,
        stick.anchor_local_pos.y
    ] if stick.anchor_local_pos else None,
}
```

---

## 9.1 为什么要记录 attached_wall_id？

因为当前 `Stick` 已经重构为：

```python
self.attached_wall
self.anchor_local_pos
```

恢复时不能直接保存对象引用。

应该保存：

```text
attached_wall_id
```

恢复时再根据 ID 找到对应墙壁对象：

```python
wall = level.get_wall_by_id(attached_wall_id)
stick.attached_wall = wall
```

---

# 10. 墙壁状态记录

需要记录每一面墙的运行时状态。

每个墙至少记录：

```python
"walls": {
    wall.id: {
        "active": wall.active,
        "isSolid": wall.isSolid,
        "solid_locked": wall.solid_locked,

        "x": wall.x,
        "y": wall.y,
        "width": wall.width,
        "height": wall.height,
        "angle": wall.angle,

        "components": {...}
    }
}
```

---

## 10.1 为什么要记录 active？

脆弱墙壁碎裂后：

```python
wall.active = False
```

如果玩家在存档点之后死亡，恢复时需要保持：

```text
存档时已经碎掉的墙仍然碎掉
存档时还没碎的墙仍然存在
```

所以必须记录：

```python
active
```

---

## 10.2 为什么要记录 isSolid？

对于实线 / 虚线墙：

```python
wall.isSolid
```

可能随着节拍或钥匙发生变化。

恢复时需要回到存档那一刻的状态。

---

## 10.3 为什么要记录 x/y？

移动墙的位置会变化。

存档点触发时，移动墙可能在路径中间。

恢复时应该让移动墙回到存档时的位置，而不是回到路径起点。

---

# 11. 组件状态记录

某些组件有运行时状态，也需要记录。

---

## 11.1 MovingComponent 状态

移动墙需要记录策略内部状态。

例如：

```python
"moving": {
    "elapsed": strategy._elapsed,
    "finished": getattr(strategy, "finished", False)
}
```

原因：

```text
只记录墙的 x/y 不够。
如果不记录 _elapsed，恢复后下一帧移动策略可能把墙跳回别的位置。
```

所以恢复时需要同时恢复：

```python
strategy._elapsed
strategy.finished
```

---

## 11.2 UnstableComponent 状态

不稳定墙需要记录：

```python
"unstable": {
    "tilt_angle": comp.tilt_angle,
    "angular_velocity": comp.angular_velocity,
    "angular_acceleration": comp.angular_acceleration,
    "attached_stick": False
}
```

注意：

```text
attached_stick 不建议直接保存对象引用。
恢复时由 stick.attached_wall 和 wall.on_anchor_attached 重新建立。
```

---

## 11.3 BeatSolidSwitchComponent 状态

节拍虚实墙需要记录：

```python
"beat_solid_switch": {
    "timer": comp.timer,
    "initial_delay": comp.initial_delay,
    "started": comp.started
}
```

否则恢复后节拍会错位。

---

## 11.4 KeyPairSolidComponent 状态

钥匙墙需要记录：

```python
"key_pair_solid": {
    "triggered": comp.triggered
}
```

如果钥匙已经触发，该墙可能已经永久变实体或虚体。

---

# 12. 道具状态记录

需要记录每个道具是否还存在。

例如：

```python
"items": {
    item.id: {
        "active": item.active,
        "x": item.x,
        "y": item.y,
        "width": item.width,
        "height": item.height,
    }
}
```

---

## 12.1 checkpoint 本身是否会消失？

不会。

存档点应该：

```python
consume_on_trigger = False
```

所以触发后仍然可见。

但可以有视觉状态：

```text
当前激活的存档点发光
旧存档点不发光
```

---

## 12.2 KeyPair 钥匙是否会恢复？

取决于存档时的状态。

如果玩家在存档前已经拿了钥匙：

```text
恢复时钥匙仍然消失
对应墙仍保持钥匙触发后的状态
```

如果玩家在存档后拿了钥匙，然后死亡：

```text
恢复到存档点时，钥匙应该回到存档时状态
如果存档时钥匙还在，则恢复后钥匙还在
```

所以必须记录 item.active。

# 13. 岩浆状态记录

岩浆至少记录：

```python
"lava": {
    "enabled": lava.enabled,
    "y": lava.y,

```


# 16. CheckpointManager

建议新增一个管理器，但不是新增实体。

例如：

```text
systems/checkpoint_manager.py
```

或者放在：

```text
level/checkpoint_manager.py
```

它负责：

```text
1. 创建 snapshot
2. 保存当前激活 checkpoint_id
3. 恢复 snapshot
4. 清空 snapshot
```

---

# 17. CheckpointManager 基本结构

```python
class CheckpointManager:
    """
    存档点管理器。

    注意：
    - 它不是地图实体。
    - 存档点实体仍然是 Item。
    - 本类只负责记录和恢复关卡运行状态。
    """

    def __init__(self):
        self.active_checkpoint_id = None
        self.snapshot = None

    def has_checkpoint(self):
        return self.snapshot is not None

    def clear(self):
        self.active_checkpoint_id = None
        self.snapshot = None
```

---

# 18. 触发存档点

```python
def activate_checkpoint(self, level, stick, checkpoint_item):
    """
    激活一个存档点。

    规则：
    - 玩家只能有一个激活存档点。
    - 激活新存档点会覆盖旧存档点。
    - checkpoint_item 本身不会被 consume。
    """

    self.active_checkpoint_id = checkpoint_item.checkpoint_id
    self.snapshot = self.create_snapshot(level, stick)

    # 可选：通知 UI / 音效 / 粒子系统
    # event_bus.emit("checkpoint_activated", {...})
```

---

# 19. 创建 snapshot

```python
def create_snapshot(self, level, stick):
    """
    创建当前关卡运行状态快照。
    """

    return {
        "checkpoint_id": self.active_checkpoint_id,
        "stick": self._snapshot_stick(stick),
        "walls": self._snapshot_walls(level.walls),
        "items": self._snapshot_items(level.items),
        "hazards": self._snapshot_hazards(getattr(level, "hazards", [])),
        "lava": self._snapshot_lava(getattr(level, "lava", None)),
        "camera": self._snapshot_camera(getattr(level, "camera", None)),
        "level_runtime": self._snapshot_runtime(level),
    }
```

# 21. 记录 Walls

```python
def _snapshot_walls(self, walls):
    data = {}

    for wall in walls:
        wall_data = {
            "active": wall.active,
            "isSolid": wall.isSolid,
            "solid_locked": wall.solid_locked,

            "x": wall.x,
            "y": wall.y,
            "width": wall.width,
            "height": wall.height,
            "angle": wall.angle,

            "components": {},
        }

        for name, comp in wall.components.items():
            if name == "moving":
                strategy = comp.move_strategy
                wall_data["components"]["moving"] = {
                    "elapsed": getattr(strategy, "_elapsed", 0.0),
                    "finished": getattr(strategy, "finished", False),
                }

            elif name == "unstable":
                wall_data["components"]["unstable"] = {
                    "tilt_angle": comp.tilt_angle,
                    "angular_velocity": comp.angular_velocity,
                    "angular_acceleration": comp.angular_acceleration,
                }

            elif name == "beat_solid_switch":
                wall_data["components"]["beat_solid_switch"] = {
                    "timer": comp.timer,
                    "initial_delay": comp.initial_delay,
                    "started": comp.started,
                }

            elif name == "key_pair_solid":
                wall_data["components"]["key_pair_solid"] = {
                    "triggered": comp.triggered,
                }

        data[wall.id] = wall_data

    return data
```

# 26. 恢复 snapshot

```python
def restore(self, level, stick):
    """
    从当前存档点恢复。

    如果没有 snapshot，则返回 False。
    """

    if self.snapshot is None:
        return False

    snap = self.snapshot

    self._restore_walls(level.walls, snap["walls"])
    self._restore_items(level.items, snap["items"])
    self._restore_hazards(getattr(level, "hazards", []), snap["hazards"])
    self._restore_lava(getattr(level, "lava", None), snap["lava"])
    self._restore_camera(getattr(level, "camera", None), snap["camera"])

    self._restore_stick(level, stick, snap["stick"])

    return True
```

注意恢复顺序：

```text
先恢复墙壁，再恢复 stick
```

因为 stick 需要根据：

```python
attached_wall_id
```

重新找到墙壁对象。

---

# 27. 恢复 Walls

```python
def _restore_walls(self, walls, data):
    wall_by_id = {wall.id: wall for wall in walls}

    for wall_id, wall_data in data.items():
        wall = wall_by_id.get(wall_id)

        if wall is None:
            continue

        wall.active = wall_data["active"]
        wall.isSolid = wall_data["isSolid"]
        wall.solid_locked = wall_data["solid_locked"]

        wall.x = wall_data["x"]
        wall.y = wall_data["y"]
        wall.width = wall_data["width"]
        wall.height = wall_data["height"]
        wall.angle = wall_data["angle"]
        wall._rect_dirty = True

        comp_data = wall_data.get("components", {})

        for name, state in comp_data.items():
            comp = wall.get_component(name)

            if comp is None:
                continue

            if name == "moving":
                strategy = comp.move_strategy
                if strategy is not None:
                    strategy._elapsed = state.get("elapsed", 0.0)
                    if hasattr(strategy, "finished"):
                        strategy.finished = state.get("finished", False)

            elif name == "unstable":
                comp.tilt_angle = state.get("tilt_angle", 0.0)
                comp.angular_velocity = state.get("angular_velocity", 0.0)
                comp.angular_acceleration = state.get("angular_acceleration", 0.0)
                comp.attached_stick = None
                comp.anchor_local_pos = None

            elif name == "beat_solid_switch":
                comp.timer = state.get("timer", 0.0)
                comp.initial_delay = state.get("initial_delay", 0.0)
                comp.started = state.get("started", True)

            elif name == "key_pair_solid":
                comp.triggered = state.get("triggered", False)
```

```python
def _restore_items(self, items, data):
    item_by_id = {item.id: item for item in items}

    for item_id, item_data in data.items():
        item = item_by_id.get(item_id)

        if item is None:
            continue

        item.active = item_data["active"]
        item.x = item_data["x"]
        item.y = item_data["y"]
        item.width = item_data["width"]
        item.height = item_data["height"]
```

# 32. 恢复 Stick

```python
def _restore_stick(self, level, stick, data):
    """
    恢复棍子状态。

    注意：
    - attached_wall 不能从 snapshot 直接保存对象。
    - 需要通过 attached_wall_id 在当前 level.walls 中查找。
    """

    stick.length = data["length"]
    stick.angular_speed = data["angular_speed"]
    stick.rotation_direction = data["rotation_direction"]

    stick.angle = data["angle"]
    stick.center_x = data["center_x"]
    stick.center_y = data["center_y"]
    stick.velocity_x = data["velocity_x"]
    stick.velocity_y = data["velocity_y"]

    stick.state = data["state"]
    stick.anchor_side = data["anchor_side"]
    stick.anchor_x = data["anchor_x"]
    stick.anchor_y = data["anchor_y"]

    stick.base_angular_speed = data.get("base_angular_speed", stick.angular_speed)
    stick.base_length = data.get("base_length", stick.length)

    # 清理旧引用
    stick.attached_wall = None
    stick.anchor_local_pos = None

    attached_wall_id = data.get("attached_wall_id")
    anchor_local_pos = data.get("anchor_local_pos")

    if attached_wall_id is not None and anchor_local_pos is not None:
        wall = level.get_wall_by_id(attached_wall_id)

        if wall is not None and wall.active:
            from physics.vector import Vector2

            stick.attached_wall = wall
            stick.anchor_local_pos = Vector2(anchor_local_pos)

            # 重新建立墙壁组件对 stick 的引用。
            wall.on_anchor_attached(
                stick,
                (stick.anchor_x, stick.anchor_y)
            )
        else:
            # 如果墙没了，则安全地变成空中状态。
            stick.state = "airborne"
```

# 34. Level 中处理 Checkpoint 道具

当检测到道具触发时：

```python
def apply_item_to_stick(self, stick, item):
    """
    应用道具效果。
    """

    if item.effect == "Checkpoint":
        self.checkpoint_manager.activate_checkpoint(
            self,
            stick,
            item
        )
        return

    if item.effect == "KeyPair":
        self.trigger_key_pair(item.key_pair_id)

        if item.consume_on_trigger:
            item.active = False

        return

    stick.apply_item(item.effect, item.value)

    if item.consume_on_trigger:
        item.active = False
```

# 35. OnTouch 触发检测

如果存档点使用：

```text
trigger_condition = OnTouch
```

需要每帧检测棍子的所有部分是否碰到 checkpoint。

伪代码：

```python
def update_item_touch_triggers(self, stick):
    """
    每帧检测 OnTouch 类型道具。
    """

    for item in self.items:
        if not item.active:
            continue

        if item.effect != "Checkpoint":
            continue

        if item.trigger_condition != "OnTouch":
            continue

        if self._stick_touches_rect(stick, item.rect):
            self.apply_item_to_stick(stick, item)
```

## 35.1 避免重复激活

由于 checkpoint 不会消失，如果每帧都触发会不断重建 snapshot。

可以给 `Item` 增加：

```python
self.activated = False
```

但因为玩家可以重新经过当前存档点刷新存档，也可以允许重复激活。

推荐规则：

```text
如果触碰的是当前激活 checkpoint，可以间隔一段时间再刷新。
如果触碰的是新的 checkpoint，立即覆盖旧 checkpoint。
```

简化版可以先允许每次触碰都刷新。

# 46. 一句话总结

存档点不是新实体类。

它应该复用：

```python
Item
```

并通过：

```python
prefab = "checkpoint"
effect = "Checkpoint"
trigger_condition = "OnTouch"
consume_on_trigger = False
```

表示。

真正负责存档和恢复的是：

```python
CheckpointManager
```

它保存当前关卡的 snapshot。

玩家死亡或按 R 时：

```text
优先恢复最近激活的 checkpoint snapshot
```

玩家按 ESC 时：

```text
清空 checkpoint，从关卡开头重新开始
```

同一时间只能有一个激活存档点，新的存档点会覆盖旧的存档记录。
