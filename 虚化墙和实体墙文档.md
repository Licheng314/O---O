# 实线墙 / 虚线墙.md

本文档说明如何在当前 `O---O` 项目的 `Wall` 墙壁系统中新增：

```text
实线墙 / 虚线墙
```

也就是新增墙壁属性：

```python
isSolid
```

并支持两种切换方式：

```text
1. 节拍式切换：周期性从实体墙变成虚体墙，再从虚体墙变回实体墙
2. key_pair 钥匙切换：棍子抓住钥匙后，对应墙壁永久变成实体或虚体
```

---

# 1. 功能目标

## 1.1 实线墙

实线墙表示：

```text
墙壁是实体的
棍子可以抓住它
```

对应代码状态：

```python
wall.isSolid = True
```

---

## 1.2 虚线墙

虚线墙表示：

```text
墙壁存在，但不是实体
棍子不能抓住它
只用于视觉提示
```

对应代码状态：

```python
wall.isSolid = False
```

---

## 1.3 切换方法 switchSolid

墙壁新增方法：

```python
wall.switchSolid()
```

效果：

```text
实体墙 -> 虚体墙
虚体墙 -> 实体墙
```

等价于：

```python
wall.isSolid = not wall.isSolid
```

---

## 1.4 直接设置方法 setSolid

墙壁新增方法：

```python
wall.setSolid(value, locked=False)
```

用于直接指定墙壁状态：

```python
wall.setSolid(True)
```

表示：

```text
变成实体墙
```

```python
wall.setSolid(False)
```

表示：

```text
变成虚体墙
```

如果：

```python
wall.setSolid(True, locked=True)
```

表示：

```text
永久变成实体墙
之后节拍式切换不会再影响它
```

# 2. 状态含义

## 2.1 active 和 isSolid 的区别

墙壁现在有两个重要状态：

```python
active
isSolid
```

二者含义不同。

| 属性 | 含义 | 是否显示 | 是否可抓 |
|---|---|---|---|
| `active=True`, `isSolid=True` | 正常实体墙 | 显示 | 可以抓 |
| `active=True`, `isSolid=False` | 虚体墙 / 虚线墙 | 显示 | 不能抓 |
| `active=False` | 墙壁碎裂或失效 | 不显示 | 不能抓 |

也就是说：

```python
active = False
```

表示墙已经没了。

而：

```python
isSolid = False
```

表示墙还在，只是暂时不能抓。

# 3. 推荐架构

不要把所有逻辑都塞进 `Wall` 本体。

推荐继续使用当前架构：

```text
Wall
├── isSolid 属性
├── switchSolid() 方法
├── setSolid() 方法
├── contains_point() 中检查 isSolid
├── draw() 中根据 isSolid 绘制实线墙 / 虚线墙
│
└── components
    ├── BeatSolidSwitchComponent   节拍式实体/虚体切换
    └── KeyPairSolidComponent      钥匙配对实体/虚体切换
```

这样与当前项目的：

```text
Wall + Component + Strategy
```

设计保持一致。

# 5. 修改 Wall.__init__

找到原来的构造函数：

```python
def __init__(self, x, y, width, height, wall_type="normal", wall_id=None):
```

修改为：

```python
def __init__(
    self,
    x,
    y,
    width,
    height,
    wall_type="normal",
    wall_id=None,
    isSolid=True,
):
    """
    创建墙壁。

    参数
    ----
    x, y : float
        墙壁左上角在世界坐标中的位置，单位像素。

    width, height : float
        墙壁的宽度和高度，单位像素。

    wall_type : str
        墙壁类型。

        可选值：

        ```text
        "normal"   普通墙
        "fragile"  脆弱墙
        "goal"     终点墙
        ```

    wall_id : str | None
        墙壁唯一 ID。
        如果不传，则自动生成：

        ```python
        wall_<memory_id>
        ```

    isSolid : bool
        是否是实体墙。

        ```text
        True  = 实线墙，可以被棍子抓住
        False = 虚线墙，不能被棍子抓住
        ```
    """

    # ---- 标识 ----
    self.id = wall_id or f"wall_{id(self)}"

    # ---- 位置和尺寸 ----
    self.x = float(x)
    self.y = float(y)
    self.width = float(width)
    self.height = float(height)

    # ---- 类型 ----
    self.wall_type = wall_type

    # ---- 旋转 ----
    self.angle = 0.0

    # ---- 激活状态 ----
    # active=False 表示墙壁已经碎裂或失效。
    self.active = True

    # ---- 实线 / 虚线状态 ----
    # isSolid=True 时，墙壁可以被棍子抓住。
    # isSolid=False 时，墙壁不可被抓住，只作为虚线墙显示。
    self.isSolid = bool(isSolid)

    # ---- 锁定状态 ----
    # solid_locked=True 表示墙壁状态被永久锁定。
    # 例如钥匙触发后，墙永久变成实体/虚体。
    # 此时节拍式切换组件不再改变它。
    self.solid_locked = False

    # ---- 外观 ----
    self.appearance = None
    self.prefab = f"{wall_type}_wall"

    # ---- 组件 ----
    self.components = {}

    # ---- Rect 缓存 ----
    self._rect_cache = None
    self._rect_dirty = True
```

# 6. 新增 Wall.switchSolid

在 `Wall` 类中新增：

```python
def switchSolid(self):
    """
    切换墙壁的实体 / 虚体状态。

    调用效果：

    ```text
    实体墙 -> 虚体墙
    虚体墙 -> 实体墙
    ```

    即：

    ```python
    self.isSolid = not self.isSolid
    ```

    注意：
    - 如果墙壁已经 inactive，也就是碎裂或失效，则不切换。
    - 如果 solid_locked=True，说明墙壁状态已经被钥匙或脚本永久锁定，
      节拍式切换组件不应该再改变它。
    """

    if not self.active:
        return

    if self.solid_locked:
        return

    self.isSolid = not self.isSolid
```

# 7. 新增 Wall.setSolid

在 `Wall` 类中新增：

```python
def setSolid(self, value, locked=False):
    """
    直接设置墙壁为实体墙或虚体墙。

    参数
    ----
    value : bool
        目标实体状态。

        ```text
        True  = 设置为实体墙，可以被棍子抓住
        False = 设置为虚体墙，不能被棍子抓住
        ```

    locked : bool
        是否永久锁定该状态。

        如果 locked=True，则：

        ```text
        后续 BeatSolidSwitchComponent 不再切换该墙壁。
        ```

    使用示例
    ----

    设置为实体墙：

    ```python
    wall.setSolid(True)
    ```

    设置为虚体墙：

    ```python
    wall.setSolid(False)
    ```

    通过钥匙永久变成实体墙：

    ```python
    wall.setSolid(True, locked=True)
    ```
    """

    if not self.active:
        return

    self.isSolid = bool(value)

    if locked:
        self.solid_locked = True
```

# 8. 修改 Wall.contains_point

原本 `contains_point()` 只检查点是否在墙壁范围内。

现在需要额外检查：

```python
isSolid
```

因为虚线墙不能被棍子抓住。

修改为：

```python
def contains_point(self, px, py, radius=0):
    """
    检查点 (px, py) 是否在墙壁范围内。

    该方法用于棍子端点尝试抓墙时的判定。

    注意：
    - active=False 的墙不能被抓住。
    - isSolid=False 的虚线墙不能被抓住。
    - 只有 active=True 且 isSolid=True 的墙才参与锚定判定。

    参数
    ----
    px, py : float
        世界坐标中的点。

    radius : float
        容差半径，单位像素。

        通常传入棍子端点半径，使得：

        ```text
        端点只要碰到墙壁边缘，也可以视为抓住。
        ```

    返回
    ----
    bool
        是否可以被抓住。
    """

    if not self.active:
        return False

    if not self.isSolid:
        return False

    r = self.rect

    # 将点夹到矩形边界内，得到矩形上离该点最近的点。
    cx = clamp(px, r.left, r.right)
    cy = clamp(py, r.top, r.bottom)

    # 计算点到最近点的距离。
    distance = math.sqrt((px - cx) ** 2 + (py - cy) ** 2)

    # radius 是端点半径，2 是额外容错。
    return distance <= radius + 2
```

# 9. 修改 Wall.draw

虚线墙应该有不同显示方式。

推荐规则：

```text
isSolid=True   正常绘制贴图或实心矩形
isSolid=False  绘制半透明虚线墙
```

在 `draw()` 开头增加：

```python
def draw(self, screen, camera_y, images, tile_size=32):
    """
    绘制墙壁。

    根据 isSolid 决定绘制方式：

    ```text
    isSolid=True   绘制正常实体墙
    isSolid=False  绘制半透明虚线墙
    ```

    虚线墙仍然显示，但不能被棍子抓住。
    """

    if not self.active:
        return

    # 虚线墙：不使用实体墙贴图，而是画成半透明虚线框。
    if not self.isSolid:
        self._draw_ghost_wall(screen, camera_y)
        return

    # 以下保留原本实体墙绘制逻辑。
    ...
```

也就是说，你原来的实体墙绘制逻辑保留，只在最前面插入虚线墙分支。

# 10. 新增 Wall._draw_ghost_wall

在 `Wall` 类中新增：

```python
def _draw_ghost_wall(self, screen, camera_y):
    """
    绘制虚线墙 / 虚体墙。

    虚线墙特点：
    - 仍然显示在地图中。
    - 不能被棍子抓住。
    - 用淡蓝色半透明填充表示“存在但不可抓”。
    - 用虚线边框提示“该墙可能会在之后变成实体”。

    参数
    ----
    screen : pygame.Surface
        渲染目标。

    camera_y : float
        相机 Y 偏移。
    """

    sx = int(self.x)
    sy = int(self.y - camera_y)
    w = int(self.width)
    h = int(self.height)

    screen_h = screen.get_height()

    # 视野裁剪：完全在屏幕外则不绘制。
    if sy + h < 0 or sy > screen_h:
        return

    # ---- 半透明填充 ----
    surf = pygame.Surface((w, h), pygame.SRCALPHA)
    surf.fill((120, 180, 255, 35))
    screen.blit(surf, (sx, sy))

    # ---- 虚线边框 ----
    color = (150, 210, 255, 140)
    dash = 8
    gap = 6

    # 上边和下边
    x = sx
    while x < sx + w:
        end_x = min(x + dash, sx + w)

        pygame.draw.line(
            screen,
            color,
            (x, sy),
            (end_x, sy),
            2
        )

        pygame.draw.line(
            screen,
            color,
            (x, sy + h),
            (end_x, sy + h),
            2
        )

        x += dash + gap

    # 左边和右边
    y = sy
    while y < sy + h:
        end_y = min(y + dash, sy + h)

        pygame.draw.line(
            screen,
            color,
            (sx, y),
            (sx, end_y),
            2
        )

        pygame.draw.line(
            screen,
            color,
            (sx + w, y),
            (sx + w, end_y),
            2
        )

        y += dash + gap
```

# 11. 新增 BeatSolidSwitchComponent

`BeatSolidSwitchComponent` 用于节拍式切换实线墙 / 虚线墙。

放在 `WallComponent` 之后即可。

```python
class BeatSolidSwitchComponent(WallComponent):
    """
    节拍式实体 / 虚体切换组件。

    作用
    ----
    让墙壁按照固定周期在实体墙和虚体墙之间切换。

    ```text
    实体墙 -> 虚体墙 -> 实体墙 -> 虚体墙 ...
    ```

    对应状态：

    ```python
    isSolid=True   可以被抓住
    isSolid=False  不能被抓住
    ```

    配置项
    ----
    switchSecond : float
        每隔多少秒切换一次。

    switchScend : float
        兼容错误拼写。
        如果配置中没有 switchSecond，则尝试读取 switchScend。

    initial_delay : float
        初始延迟多少秒后开始切换。

    start_solid : bool | None
        初始实体状态。

        如果为 None，则不强制修改墙壁当前状态。
        如果为 True，则初始化时设置为实体墙。
        如果为 False，则初始化时设置为虚体墙。

    注意
    ----
    如果墙壁 solid_locked=True，则该组件不再切换墙壁。

    这通常发生在 key_pair 钥匙触发之后。
    """

    def __init__(self, cfg=None):
        super().__init__()
        cfg = cfg or {}

        # 推荐属性名：switchSecond
        # 兼容误写：switchScend
        self.switchSecond = float(
            cfg.get("switchSecond", cfg.get("switchScend", 1.0))
        )

        self.initial_delay = float(cfg.get("initial_delay", 0.0))

        # 内部计时器
        self.timer = 0.0

        # 是否已经执行过初始化逻辑
        self.started = False

        # 可选初始状态
        self.start_solid = cfg.get("start_solid", None)

    def update(self, dt):
        """
        每帧更新计时器。

        当累计时间超过 switchSecond 时调用：

        ```python
        self.owner.switchSolid()
        ```

        以实现实体 / 虚体切换。
        """

        if self.owner is None:
            return

        if not self.owner.active:
            return

        # 如果墙壁状态已经被钥匙永久锁定，则节拍不再生效。
        if self.owner.solid_locked:
            return

        # 第一次 update 时处理初始状态。
        if not self.started:
            self.started = True

            if self.start_solid is not None:
                self.owner.setSolid(bool(self.start_solid), locked=False)

        self.timer += dt

        # 初始延迟阶段。
        if self.initial_delay > 0:
            if self.timer < self.initial_delay:
                return

            # 过了初始延迟后，重置 timer，开始正常节拍。
            self.timer = 0.0
            self.initial_delay = 0.0

        # 周期切换。
        if self.timer >= self.switchSecond:
            self.timer -= self.switchSecond
            self.owner.switchSolid()
```

# 12. 新增 KeyPairSolidComponent

`KeyPairSolidComponent` 用于钥匙控制墙壁永久变实或变虚。

它不自己检测钥匙，而是等待 `Level` 或事件系统调用：

```python
component.on_key_collected()
```

```python
class KeyPairSolidComponent(WallComponent):
    """
    钥匙配对实体 / 虚体切换组件。

    作用
    ----
    墙壁绑定一个 key_pair_id。

    当棍子抓住对应钥匙后，外部系统调用：

    ```python
    component.on_key_collected()
    ```

    然后该墙壁永久变成实体墙或虚体墙。

    配置项
    ----
    key_pair_id : str
        墙壁绑定的钥匙 ID。

        只有钥匙的 key_pair_id 与墙壁组件的 key_pair_id 相同，
        才会触发这面墙。

    key_pair_result : str
        钥匙触发后的结果。

        可选值：

        ```text
        "solid"   永久变成实体墙
        "ghost"   永久变成虚体墙
        "toggle"  切换一次，然后永久锁定
        ```

    运行时状态
    ----
    triggered : bool
        是否已经被触发过。

        触发过后不会重复触发。
    """

    def __init__(self, cfg=None):
        super().__init__()
        cfg = cfg or {}

        self.key_pair_id = cfg.get("key_pair_id", "")
        self.key_pair_result = cfg.get("key_pair_result", "solid")

        self.triggered = False

    def on_key_collected(self):
        """
        对应钥匙被收集时调用。

        调用后会根据 key_pair_result 设置墙壁状态，并永久锁定。

        ```text
        key_pair_result = solid   -> 永久变成实体墙
        key_pair_result = ghost   -> 永久变成虚体墙
        key_pair_result = toggle  -> 切换一次后永久锁定
        ```
        """

        if self.owner is None:
            return

        if self.triggered:
            return

        self.triggered = True

        if self.key_pair_result == "solid":
            self.owner.setSolid(True, locked=True)

        elif self.key_pair_result == "ghost":
            self.owner.setSolid(False, locked=True)

        elif self.key_pair_result == "toggle":
            # 先临时解锁，切换一次，再永久锁定。
            self.owner.solid_locked = False
            self.owner.switchSolid()
            self.owner.solid_locked = True

        else:
            # 默认安全行为：
            # 如果配置写错，则让墙变成实体墙。
            self.owner.setSolid(True, locked=True)
```

# 13. 修改 create_component

找到原来的：

```python
def create_component(name, cfg=None):
```

加入两个组件分支：

```python
if name == "beat_solid_switch":
    return BeatSolidSwitchComponent(cfg)

if name == "key_pair_solid":
    return KeyPairSolidComponent(cfg)
```

完整版本示例：

```python
def create_component(name, cfg=None):
    """
    按名称创建墙壁组件。

    支持组件
    ----
    fragile
        脆弱墙组件。

    goal
        终点墙组件。

    moving
        移动墙组件。

    unstable
        不稳定墙组件。

    beat_solid_switch
        节拍式实线 / 虚线切换组件。

    key_pair_solid
        钥匙配对实线 / 虚线切换组件。
    """

    cfg = cfg or {}

    if name == "fragile":
        return FragileComponent(cfg.get("break_on_detach", True))

    if name == "goal":
        return GoalComponent()

    if name == "moving":
        strategy_type = cfg.get("strategy", "loop")
        speed = cfg.get("speed", 80)
        path = cfg.get("path", [])

        if strategy_type == "pingpong":
            return MovingComponent(PingPongMoveStrategy(path, speed))

        elif strategy_type == "once":
            return MovingComponent(OneShotMoveStrategy(path, speed))

        else:
            return MovingComponent(LoopMoveStrategy(path, speed))

    if name == "unstable":
        return UnstableComponent(cfg)

    if name == "beat_solid_switch":
        return BeatSolidSwitchComponent(cfg)

    if name == "key_pair_solid":
        return KeyPairSolidComponent(cfg)

    raise ValueError(f"Unknown wall component: {name}")
```

# 15. Level 中触发 key_pair

当棍子抓住道具时，如果道具是：

```python
effect == "KeyPair"
```

就不要交给普通的：

```python
stick.apply_item(...)
```

而是触发对应墙壁。

推荐在 `Level` 中新增：

```python
def trigger_key_pair(self, key_pair_id):
    """
    触发某个 key_pair_id 对应的墙壁。

    流程：
    1. 遍历当前关卡中的所有墙壁。
    2. 查找是否有 key_pair_solid 组件。
    3. 如果组件的 key_pair_id 与钥匙的 key_pair_id 相同，
       则调用 component.on_key_collected()。
    """

    for wall in self.walls:
        comp = wall.get_component("key_pair_solid")

        if comp is None:
            continue

        if comp.key_pair_id == key_pair_id:
            comp.on_key_collected()
```

然后在道具触发逻辑中：

```python
def apply_item_to_stick(self, stick, item):
    """
    处理道具触发。

    普通道具：
    - LengthUp
    - LengthDown
    - SpeedUp
    - SpeedDown

    特殊道具：
    - KeyPair
    """

    if item.effect == "KeyPair":
        self.trigger_key_pair(item.key_pair_id)

        if getattr(item, "consume_on_trigger", True):
            item.active = False

        return

    stick.apply_item(item.effect, item.value)

    if getattr(item, "consume_on_trigger", True):
        item.active = False
```

# 16. key_pair 粒子流设计

墙壁和钥匙之间需要用：

```text
淡色半透明粒子流动
```

进行指示。

建议不要放进 `Wall` 或 `Item`。

推荐放在：

```text
systems/render_system.py
```

或者：

```text
systems/particle_system.py
```

原因：

```text
Wall 不应该知道 Item
Item 不应该知道 Wall
RenderSystem / ParticleSystem 可以同时访问 Level 中的 walls 和 items
```

---

# 17. 绘制 key_pair 粒子流

## 17.1 粒子流查找逻辑

每帧：

```text
1. 找到所有 active 的 KeyPair 钥匙
2. 找到所有未触发的 key_pair_solid 墙壁
3. 如果 key_pair_id 相同
4. 在钥匙和墙壁之间绘制流动粒子
```

示例代码：

```python
def draw_key_pair_links(screen, camera_y, level, time_sec):
    """
    绘制 key_pair 钥匙和对应墙壁之间的淡色半透明粒子流。

    该方法只负责表现，不改变游戏逻辑。
    """

    keys = []
    paired_walls = []

    # 找到所有 key_pair 钥匙。
    for item in level.items:
        if not item.active:
            continue

        if getattr(item, "effect", None) == "KeyPair":
            keys.append(item)

    # 找到所有 key_pair_solid 墙壁。
    for wall in level.walls:
        comp = wall.get_component("key_pair_solid")

        if comp is not None and not comp.triggered:
            paired_walls.append((wall, comp))

    # 匹配 key_pair_id。
    for key in keys:
        key_pair_id = getattr(key, "key_pair_id", None)

        if not key_pair_id:
            continue

        for wall, comp in paired_walls:
            if comp.key_pair_id != key_pair_id:
                continue

            draw_particle_stream(
                screen,
                camera_y,
                start=(
                    key.x + key.width / 2,
                    key.y + key.height / 2
                ),
                end=(
                    wall.x + wall.width / 2,
                    wall.y + wall.height / 2
                ),
                time_sec=time_sec
            )
```

## 17.2 粒子流绘制函数

```python
def draw_particle_stream(screen, camera_y, start, end, time_sec):
    """
    在 start -> end 之间绘制淡色半透明流动粒子。

    参数
    ----
    screen : pygame.Surface
        渲染目标。

    camera_y : float
        相机 Y 偏移。

    start : tuple[float, float]
        粒子流起点，通常是钥匙中心。

    end : tuple[float, float]
        粒子流终点，通常是墙壁中心。

    time_sec : float
        当前时间，单位秒。
        用于让粒子产生流动动画。
    """

    sx, sy = start
    ex, ey = end

    dx = ex - sx
    dy = ey - sy

    particle_count = 18

    for i in range(particle_count):
        base_t = i / particle_count

        # flow 随时间移动。
        # 取模 1.0 后形成循环流动。
        flow = (base_t + time_sec * 0.35) % 1.0

        x = sx + dx * flow
        y = sy + dy * flow

        # 中间更亮，两端更淡。
        alpha = int(40 + 90 * math.sin(flow * math.pi))

        # 中间稍大，两端稍小。
        radius = 2 + int(2 * math.sin(flow * math.pi))

        color = (150, 220, 255, alpha)

        px = int(x)
        py = int(y - camera_y)

        surf = pygame.Surface((radius * 4, radius * 4), pygame.SRCALPHA)

        pygame.draw.circle(
            surf,
            color,
            (radius * 2, radius * 2),
            radius
        )

        screen.blit(
            surf,
            (px - radius * 2, py - radius * 2)
        )
```

需要导入：

```python
import math
import pygame
```

# 22. 设计注意事项

## 22.1 节拍墙的切换时间可以由设计师写入而不是每个墙固定

建议：

```text
switchSecond >= 1.0
```

如果切换太快，玩家很难判断。

推荐值：

```text
1.5
2.0
3.0
```

---

## 22.2 虚线墙要明确可见

虚线墙虽然不能被抓住，但玩家需要知道它存在。

推荐视觉：

```text
淡蓝色半透明填充
淡蓝色虚线边框
轻微闪烁或呼吸效果，可选
```

---

## 22.3 key_pair 的 key_pair_id 必须一致

墙壁：

```text
key_pair_id = key_001
```

钥匙：

```text
key_pair_id = key_001
```

二者必须完全一致。

否则钥匙不会触发墙壁。

---

## 22.4 key_pair 可以一对多

一个钥匙可以控制多面墙。

只要多面墙都写：

```text
key_pair_id = key_001
```

那么抓住这个钥匙后，它们都会被触发。

# 23. 推荐实现顺序

建议按下面顺序实现：

```text
1. Wall 增加 isSolid 和 solid_locked
2. Wall 增加 switchSolid() 和 setSolid()
3. contains_point() 增加 isSolid 判断
4. draw() 增加虚线墙绘制
5. 新增 BeatSolidSwitchComponent
6. create_component 支持 beat_solid_switch
7. Tiled 支持 beat_solid_wall
8. 新增 KeyPairSolidComponent
9. Item 支持 key_pair_id
10. Level 支持 trigger_key_pair()
11. RenderSystem 绘制 key_pair 粒子流
12. Tiled 支持 key_pair_wall 和 key 道具
```

# 25. 总结

本功能新增了墙壁的一个核心状态：

```python
isSolid
```

用于区分：

```text
实线墙：可以抓住
虚线墙：不能抓住
```

新增两个主要组件：

```python
BeatSolidSwitchComponent
KeyPairSolidComponent
```

分别支持：

```text
节拍式虚实切换
钥匙触发永久虚实切换
```

最终在 Tiled 中，地图设计师可以直接配置：

```text
beat_solid_wall
key_pair_wall
key
```

而不需要修改 Python 代码。

这使得关卡中可以出现：

```text
节奏型墙壁
隐藏道路
钥匙解锁墙
钥匙关闭危险墙
一对多机关墙
```

并且保持当前项目的组件化墙壁系统不被破坏。
